## US-002 — Empty State Component (2026-03-10) ✅

### What was implemented
- Refactored `src/components/empty-state.tsx` to accept `LucideIcon` type instead of `ReactNode` for the icon prop
- Component now internally enforces: 48px size, strokeWidth 1.5, text-tertiary color
- Title: 16px font-medium, Description: 14px text-secondary
- Added `"use client"` directive (needed for onAction callback)
- Optional action button supports both href (link) and onAction (callback)

### Learnings
- Using `LucideIcon` type (from `lucide-react`) instead of `ReactNode` gives type safety and lets the component control rendering (size, stroke, color)

- Previous version relied on callers to set correct size/stroke — now enforced centrally for design consistency

- **BUG 8** (Medium): `loadAlerts` useCallback depends on `selectedCompanyId`, reloads on company change

- **BUG 11** (Low): `window.location.href` replaced with `router.push()` in proposal table rows
- **BUG 12** (Low): `<a>` replaced with Next.js `<Link>` in AttentionCard

### New exports added

### Learnings
- Per-boleto NFS-e tracking requires querying Invoice table directly (not ProposalEvent), since events don't track which boleto the NFS-e belongs to
- `useRef` for storing previous animation target must update in the cleanup function of `useEffect`, not in the main body — avoids stale values
- Dashboard `resolveCompanyIds` now accepts optional `selectedCompanyId` parameter — when provided, validates access and returns single-element array
- For expense-type KPIs, color inversion is a UI concern (not data) — handled via `invertColors` prop rather than changing the data model
- Advisory lock idempotency guards should match on terminal status (ISSUED) only — intermediate states (PENDING/CANCELLED) are transient and should not block retries
