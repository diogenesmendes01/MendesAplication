### Schema atual (AiConfig)
```prisma
model AiConfig {
  id                 String   @id @default(cuid())
  companyId          String   @unique
  enabled            Boolean  @default(false)
  persona            String   @db.Text
  welcomeMessage     String?  @db.Text
  escalationKeywords String[]
  maxIterations      Int      @default(5)
  createdAt          DateTime @default(now())
  updatedAt          DateTime @updatedAt
  company Company @relation(...)
}
```

## US-SAN-002 — SantanderAuthManager — mTLS + OAuth 2.0 com cache de token (2026-03-19)

### Lições
- ⏳ Node.js fetch (undici) aceita `agent` como propriedade não-tipada — precisa de @ts-expect-error para mTLS
- ⏳ O endpoint OAuth do Santander usa application/x-www-form-urlencoded (padrão OAuth), não JSON
- ⏳ TOKEN_RENEWAL_MARGIN_MS = 60s garante que requests em andamento não usem token prestes a expirar
- ⏳ O https.Agent é singleton na instância do AuthManager — evita criar conexões TLS desnecessárias
