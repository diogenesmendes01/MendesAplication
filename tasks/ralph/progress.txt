# Ralph Progress — MendesERP (erp-multi-empresa)

## Codebase Patterns
- Project lives in `/workspaces/app/erp/` — all npm/npx commands must run from there
- Next.js 14.2.35 with App Router, TypeScript strict mode
- TailwindCSS 3.x with Shadcn/UI (New York style, default config)
- Prisma 6.x with `prisma-client-js` generator (classic, not new `prisma-client`)
- Prisma singleton in `src/lib/prisma.ts` — import from `@/lib/prisma`
- Shadcn/UI components in `src/components/ui/` — use Sonner for toasts (not the old Toaster)
- Quality checks: `npx tsc --noEmit` for typecheck, `npm run lint` for ESLint
- `.env` is gitignored; `.env.example` has the template
- Working directory for git commands: `/workspaces/app/` (root repo)
- Prisma seed config goes in `package.json` under `"prisma": { "seed": "npx tsx prisma/seed.ts" }` (prisma.config.ts `seed` not supported in Prisma 6.19)
- PostgreSQL needs manual setup: `sudo apt-get install postgresql`, `sudo su -c "..." postgres` to configure
- Use `upsert` in seed scripts for idempotent re-runs
- Shared enums (Status, UserRole) are defined at schema level and reused across models
- Auth utilities in `src/lib/auth.ts` — import `hashPassword`, `comparePassword`, `generateAccessToken`, `generateRefreshToken`, `verifyAccessToken`, `getCurrentUser`
- Auth middleware `withAuth` in `src/lib/auth-middleware.ts` — wraps API handlers with JWT validation
- Refresh token stored in httpOnly cookie "refreshToken", refresh endpoint at POST `/api/auth/refresh`
- Access token stored in non-httpOnly cookie "accessToken" so middleware can check auth state
- Next.js middleware at `src/middleware.ts` handles auth redirects: unauthenticated → /login
- Login API at POST `/api/auth/login` — validates credentials, sets both cookies
- Label component in `src/components/ui/label.tsx` for form labels
- Authenticated pages use `(app)` route group — layout at `src/app/(app)/layout.tsx`
- Sidebar component at `src/components/sidebar.tsx`, Header at `src/components/header.tsx`
- Navigation items defined in `navItems` array in sidebar.tsx — add new nav links there
- Root page `/` redirects to `/dashboard`
- Server actions use `"use server"` directive and live alongside their pages (e.g. `configuracoes/empresas/actions.ts`)
- Session helpers in `src/lib/session.ts`: `getSession()`, `requireSession()`, `requireAdmin()` — read accessToken cookie for auth in server actions
- CNPJ validation in `src/lib/cnpj.ts`: `isValidCnpj()`, `stripCnpj()`, `formatCnpj()` — validates check digits
- Server actions use `PaginatedResult<T>` type for paginated listings (page, pageSize, total, totalPages, data)
- CNPJs are stored formatted (XX.XXX.XXX/XXXX-XX) in the database
- Listing pages pattern: client component with useCallback for loadData, useEffect to trigger, search resets to page 1
- Form dialogs: reuse same Dialog for create/edit, distinguish via `editingId` state (null = create, string = edit)
- Company context: `useCompany()` hook from `@/contexts/company-context` gives `selectedCompanyId`, `selectedCompany`, `companies`
- Company-scoped server actions should accept `companyId` param and validate access (not rely on client-side context alone)
- Shadcn DropdownMenu available in `src/components/ui/dropdown-menu.tsx`
- RBAC helpers in `src/lib/rbac.ts`: `canAccessCompany()`, `canAccessModule()`, `requireCompanyAccess()`, `requireModuleAccess()` — use in company-scoped server actions
- Admin role bypasses all RBAC checks; Manager role requires UserCompany assignment and module permissions
- For company-scoped server actions, use `requireCompanyAccess(companyId)` instead of `requireSession()` — it validates both auth and company access
- User server actions in `src/app/(app)/configuracoes/usuarios/actions.ts` — exports `createUser`, `updateUser`, `listUsers`, `getUserById`, `assignUserToCompanies`, `toggleUserStatus`
- Always use explicit `select` on User queries to exclude `passwordHash` from responses
- `assignUserToCompanies` replaces all assignments atomically — pass empty array to remove all
- Shadcn Checkbox in `src/components/ui/checkbox.tsx`, Select in `src/components/ui/select.tsx` — available for forms
- AVAILABLE_MODULES for RBAC UI: "comercial", "sac", "financeiro", "fiscal", "configuracoes"
- Two-dialog pattern: separate create/edit dialog from assignment/config dialog for complex entities
- Audit logging: `logAuditEvent()` from `@/lib/audit` — fire-and-forget, never throws. Use `Prisma.InputJsonValue` for dataBefore/dataAfter types
- Logout API at POST `/api/auth/logout` — clears both cookies, logs LOGOUT audit event
- Audit log server actions in `src/app/(app)/configuracoes/auditoria/actions.ts` — `listAuditLogs`, `exportAuditLogsCsv`, `getAuditEntityTypes`, `getAuditUsers`, `getAuditCompanies`
- CSV export pattern: generate CSV string server-side, create Blob with BOM (`\uFEFF`) client-side for proper Excel encoding, trigger download via temporary `<a>` element
- Filter dropdowns: use `__all__` as sentinel value for "all items" since Radix Select doesn't support empty string values
- Dashboard data in `src/app/(app)/dashboard/actions.ts` — uses placeholder KPIs until financial tables exist; replace `generatePlaceholderKPI` with real queries after US-017+
- Currency formatting: use `Intl.NumberFormat("pt-BR", { style: "currency", currency: "BRL" })` for BRL values
- Recharts installed for charts — use `ResponsiveContainer` + `BarChart`/`LineChart` for responsive charts; wrap in a fixed-height div
- Chart currency formatting: use abbreviated format (R$ 10k, R$ 1.5M) for Y-axis labels via custom `tickFormatter`
