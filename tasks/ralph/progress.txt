# Ralph Progress — MendesERP (whatsapp-baileys-ai-agent)

## Codebase Patterns
- Project lives in `/workspaces/app/erp/` — all npm/npx commands must run from there
- WhatsApp Service lives in `/workspaces/app/whatsapp-service/` — separate Express microservice
- Next.js 14.2.35 with App Router, TypeScript strict mode
- TailwindCSS 3.x with Shadcn/UI (New York style, default config)
- Prisma 6.x with `prisma-client-js` generator (classic, not new `prisma-client`)
- Prisma singleton in `src/lib/prisma.ts` — import from `@/lib/prisma`
- Shadcn/UI components in `src/components/ui/` — use Sonner for toasts (not the old Toaster)
- Quality checks: `npx tsc --noEmit` for typecheck, `npm run lint` for ESLint
- PostgreSQL needs `sudo service postgresql start` before db operations in devcontainer
- Prisma compound unique with nullable fields: use `findFirst` + `create` instead of `upsert`
- Prisma JSON fields need `as Prisma.InputJsonValue` cast for Record<string, unknown>
- `.env` is gitignored; `.env.example` has the template
- Working directory for git commands: `/workspaces/app/` (root repo)
- Prisma seed config goes in `package.json` under `"prisma": { "seed": "npx tsx prisma/seed.ts" }`
- Use `upsert` in seed scripts for idempotent re-runs
- Shared enums (Status, UserRole) are defined at schema level and reused across models
- Auth utilities in `src/lib/auth.ts` — import `hashPassword`, `comparePassword`, `generateAccessToken`, `generateRefreshToken`, `verifyAccessToken`, `getCurrentUser`
- Session helpers in `src/lib/session.ts`: `getSession()`, `requireSession()`, `requireAdmin()`
- RBAC helpers in `src/lib/rbac.ts`: `canAccessCompany()`, `requireCompanyAccess()`
- Server actions use `"use server"` directive and live alongside their pages (e.g. `sac/tickets/actions.ts`)
- Company context: `useCompany()` hook from `@/contexts/company-context` gives `selectedCompanyId`, `selectedCompany`, `companies`
- BullMQ queues defined in `src/lib/queue.ts`, workers in `src/lib/workers/`
- Redis at localhost:6379 for BullMQ
- WhatsApp Service and ERP share same PostgreSQL database
- WhatsApp Service communicates with ERP via HTTP webhooks (localhost)

- Baileys v7 requires Node.js 20+ — devcontainer has Node 18 by default, need `sudo npx -y n 20` to upgrade
- WhatsApp Service uses ESM (`"type": "module"` in package.json) and `moduleResolution: "bundler"` in tsconfig
- When schema.prisma changes in ERP, must copy to whatsapp-service/prisma/ and run `npx prisma generate` there too
