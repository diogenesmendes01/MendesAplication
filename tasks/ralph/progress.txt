# Ralph Progress — MendesERP (erp-multi-empresa)

## Codebase Patterns
- Project lives in `/workspaces/app/erp/` — all npm/npx commands must run from there
- Next.js 14.2.35 with App Router, TypeScript strict mode
- TailwindCSS 3.x with Shadcn/UI (New York style, default config)
- Prisma 6.x with `prisma-client-js` generator (classic, not new `prisma-client`)
- Prisma singleton in `src/lib/prisma.ts` — import from `@/lib/prisma`
- Shadcn/UI components in `src/components/ui/` — use Sonner for toasts (not the old Toaster)
- Quality checks: `npx tsc --noEmit` for typecheck, `npm run lint` for ESLint
- `.env` is gitignored; `.env.example` has the template
- Working directory for git commands: `/workspaces/app/` (root repo)
- Prisma seed config goes in `package.json` under `"prisma": { "seed": "npx tsx prisma/seed.ts" }` (prisma.config.ts `seed` not supported in Prisma 6.19)
- PostgreSQL needs manual setup: `sudo apt-get install postgresql`, `sudo su -c "..." postgres` to configure
- Use `upsert` in seed scripts for idempotent re-runs
- Shared enums (Status, UserRole) are defined at schema level and reused across models
- Auth utilities in `src/lib/auth.ts` — import `hashPassword`, `comparePassword`, `generateAccessToken`, `generateRefreshToken`, `verifyAccessToken`, `getCurrentUser`
- Auth middleware `withAuth` in `src/lib/auth-middleware.ts` — wraps API handlers with JWT validation
- Refresh token stored in httpOnly cookie "refreshToken", refresh endpoint at POST `/api/auth/refresh`
- Access token stored in non-httpOnly cookie "accessToken" so middleware can check auth state
- Next.js middleware at `src/middleware.ts` handles auth redirects: unauthenticated → /login
- Login API at POST `/api/auth/login` — validates credentials, sets both cookies
- Label component in `src/components/ui/label.tsx` for form labels
